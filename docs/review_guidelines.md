# 海克斯乱斗实验室 — 审核规范

本文档定义了 AI 候选数据的人工审核标准，供审核人员在使用 `review_candidates.py` 时参考。

## Bug 审核标准

Bug 类候选（`bug_report`）来自 AI 对论坛帖子的结构化提取，描述某个英雄与增强组合的异常行为。审核时应关注以下要点：

**通过（approve）条件：**

- 描述的现象在至少一个公开来源（Reddit 帖子、视频、官方说明）中有提及
- 涉及英雄和增强均存在于正式数据中（`champions.json` / `augments.json`）
- 描述具有可复现性，而非个例误判
- 严重度（severity）与描述的影响范围匹配：`critical` 用于影响游戏平衡或数值计算错误的问题，`major` 用于明确的功能异常，`minor` 用于表现层或低概率问题

**拒绝（reject）条件：**

- AI 误报：描述的现象实际为正常机制（如伤害衰减被误认为 Bug）
- 已在正式数据 `issues.json` 中存在相同条目
- 描述过于模糊，无法验证也无法利用

**编辑（edit）建议：**

- 调整 severity 使其与实际影响匹配
- 补充或修正描述中的不准确信息
- 添加规避建议（tip）

## 质变组合审核标准

质变组合（`rating_type = transform`）是项目的核心数据，审核标准最为严格。

**通过条件：**

- 有至少一条外部证据（Reddit 讨论、视频演示、数据截图）支持该组合存在显著效果
- 英雄和增强均存在于正式数据中
- 描述清晰说明了协同机制（为什么这个组合能产生质变）
- 有明确的触发条件

**证据为空时的处理：**

如果候选没有证据链接但仍然值得保留，审核工具会自动执行以下保护：

- `status` 强制设为 `investigating`
- `confidence` 如果高于 60 则自动降至 60
- `source_note` 标注"暂无外部证据链接，需继续验证"

这类条目后续需补充证据后才能提升到 `community` 或 `verified` 状态。

**拒绝条件：**

- 英雄与增强之间没有明显的协同机制（如纯物理英雄 + 法力增强类增强）
- 效果描述夸大或与实际机制不符

## 避雷组合审核标准

避雷组合（`rating_type = trap_warning`，写入正式数据时 tier 为 `avoid`）描述某个英雄与增强搭配效果极差或存在冲突。

**通过条件：**

- 英雄确实缺少利用该增强所需的机制（如无治疗/护盾技能的英雄 + 治疗增幅类增强）
- 或者该增强与英雄存在已知的负面交互（如风墙阻挡己方弹射效果）
- 描述清楚说明了为什么应该避开这个组合

**特殊处理：**

- `trap_warning` 在写入 `synergies.json` 时自动转换为 `tier: "avoid"`
- 避雷组合即使没有外部证据，只要机制分析合理也值得保留（因为帮助玩家避免踩坑的价值较高）

## 状态使用规则

每条数据都有一个 `status` 字段，表示其可信度层级。各状态的使用场景如下：

**investigating（调查中）** — 默认状态。用于以下场景：新审核通过但尚未充分验证的条目；证据不足或证据质量较低的条目；需要进一步测试或等待更多社区反馈的条目。这是所有无证据 approve 的强制状态。

**community（社区验证中）** — 用于有多个社区来源支持但尚未经过系统性验证的条目。典型场景：Reddit 上多名玩家讨论过同一现象，但没有数据截图或官方确认。confidence 建议范围 50-75%。

**verified（已验证）** — 用于经过交叉验证的可靠条目。需要满足至少一个条件：有 Riot 官方确认（如官方论坛回复、补丁说明）；有可复现的数据截图或视频证据；有多个独立来源交叉印证。注意：verified 不等于 Riot 官方确认，仅代表在本项目内部经过了较充分的验证。confidence 建议范围 75-100%。

**disputed（争议）** — 用于社区意见分歧的条目。典型场景：部分玩家认为有效，另一部分认为无效或效果被夸大；不同测试环境下结果不一致。confidence 建议范围 30-55%。

**fixed（已修复）** — 仅用于 Bug 类条目，表示 Riot 已在某个补丁中修复了该问题。更新 status 为 fixed 时，应同时更新 `ver` 字段为修复版本号。

## evidence 类型说明

每条证据（evidence）是一个对象，包含三个字段：

| 字段 | 说明 |
|------|------|
| `type` | 证据来源类型 |
| `url` | 链接地址 |
| `summary` | 一句话摘要（可选） |

允许的 `type` 值：

**reddit** — Reddit 帖子或评论链接。最常见的证据来源，通常来自 r/ARAM、r/summonerschool 等子版块。

**video** — YouTube、Bilibili 等视频平台链接。适用于展示游戏内表现的视频证据，如 Bug 复现、伤害数值演示。

**screenshot** — 图片链接（如 Imgur、Reddit 图片帖）。适用于截图证据，如游戏内伤害统计面板、增强选择界面截图。

**official** — Riot 官方来源，包括官方论坛帖子、补丁说明页面、Riot 员工社交媒体发言。这是最高可信度的证据类型。

**manual** — 人工标注。审核人员手动添加的证据说明，通常用于标注"本人实测确认"或"基于机制分析推断"等情况，URL 可留空。

## 审核流程建议

1. 先运行 pipeline 采集最新候选数据
2. 优先审核 `confidence >= 70` 的高置信度候选
3. 对有证据的候选优先 approve，无证据的候选补充证据后再审核
4. 无证据但机制合理的候选可 approve（工具会自动保护状态和置信度）
5. 拒绝的候选应填写原因，便于后续排查 AI 提取质量
6. 定期审核 `rejected_candidates.json`，检查是否有误拒的条目
